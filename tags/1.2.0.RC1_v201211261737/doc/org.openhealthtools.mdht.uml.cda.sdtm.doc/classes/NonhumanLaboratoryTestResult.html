Non-Human Laboratory Test Result

[Observation: templateId 2.16.840.1.113883.10.20.23.32]

Information on laboratory data collected on a reporting form or received from a central provider or vendor.

  1. SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
  2. SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
  3. SHALL contain exactly one [1..1] code
    • A coded value that identifies the laboratory test result. The verbatim text is placed within the original text property of the CD data type. [SEND: LBTESTCD, LBTEST]
  4. SHALL contain [1..2] id
    • A value that is provided to ensure the uniqueness of the laboratory result. [SEND: LBSEQ, LBSPID] A single identifier, the sequence number, is required.
  5. MAY contain zero or one [0..1] statusCode
    • Used to indicate whether or not an exam has been performed. [SDTM: LBSTAT]
  6. SHOULD contain zero or one [0..1] value with data type CD
    • The laboratory result, using both original units and a standard format or set of units. [SEND: LBORRES, LBORRESU, LBSTRESC, LBSTRESN, LBSTRESU]
  7. MAY contain zero or one [0..1] entryRelationship
    • Allows capture of a group identifier for the laboratory result.
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Group Identifier (templateId: 2.16.840.1.113883.10.20.23.63)
  8. SHOULD contain zero or one [0..1] methodCode
    • Indicates the method of the test or examination. [SEND: LBMETHOD]
  9. SHOULD contain zero or one [0..1] entryRelationship
    • Used to capture information about the specimen used as the direct subject of the test.
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Specimen Information (templateId: 2.16.840.1.113883.10.20.23.67)
  10. MAY contain zero or one [0..1] entryRelationship
    • Captures the reason for not performing the indicated test.
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Non Performance Reason (templateId: 2.16.840.1.113883.10.20.23.64)
  11. MAY contain zero or one [0..1] participant
    • Used to capture the organization performing the test.
    1. Contains exactly one [1..1] Study Test Organization (templateId: 2.16.840.1.113883.10.20.23.73)
  12. MAY contain zero or one [0..1] entryRelationship

    Used to indicate whether or not the study subject was fasting at the time of specimen collection.

    1. This entryRelationship SHALL contain exactly one [1..1] @typeCode="COMP"
    2. This entryRelationship SHALL contain zero or one [0..1] observation
      1. This observation SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
      2. This observation SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
      3. This observation SHALL contain exactly one [1..1] code, where the @code SHALL be selected from (CodeSystem: 2.16.840.1.113883.3.26.1.1 NCI Thesaurus)

        The code value indicates the observation concerns the subject's fasting status.

      4. This observation SHALL contain exactly one [1..1] value with data type BN

        Information on whether the person was fasting at the time the specimen was collected. [SEND: LBFAST, PCFAST]

  13. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Exclusion Reason (templateId: 2.16.840.1.113883.10.20.23.76)
  14. SHOULD contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Planned Study Day (templateId: 2.16.840.1.113883.10.20.23.65)
  15. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Study Day Period (templateId: 2.16.840.1.113883.10.20.23.70)
  16. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Timing Reference (templateId: 2.16.840.1.113883.10.20.23.74)
  17. SHOULD contain [0..4] interpretationCode
    • Includes information used to better interpret the observation value (result). [SEND:LBBLFL, LBDRVFL, LBEXCLFL, LBNRIND]
  18. MAY contain zero or more [0..*] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Comment (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  19. SHALL contain exactly one [1..1] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Domain Assignment (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  20. MAY contain zero or more [0..*] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Related Record (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  21. MAY contain zero or more [0..*] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Supplemental Value (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  22. Contains zero or one [0..1] entryRelationship
    1. Contains exactly one [1..1] Toxicity (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  23. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Category (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  24. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Sub- Category (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  25. SHOULD contain zero or one [0..1] reference Range
    1. Contains exactly one [1..1] Reference Range (templateId: 2.16.840.1.113883.10.20.23.xxxx)
