Continuity Of Care Document

[ClinicalDocument: templateId 2.16.840.1.113883.10.20.1]

  1. SHALL contain exactly one [1..1] templateId ( CONF-7, CONF-8 ) such that it
    1. SHALL contain exactly one [1..1] @root="2.16.840.1.113883.10.20.1"
  2. SHOULD contain exactly one [1..1] component (CONF-140)
    1. Contains exactly one [1..1] Problem Section (templateId: 2.16.840.1.113883.10.20.1.11)
  3. SHOULD contain exactly one [1..1] component (CONF-184)
    1. Contains exactly one [1..1] Family History Section (templateId: 2.16.840.1.113883.10.20.1.4)
  4. SHOULD contain exactly one [1..1] component (CONF-232)
    1. Contains exactly one [1..1] Social History Section (templateId: 2.16.840.1.113883.10.20.1.15)
  5. SHOULD contain exactly one [1..1] component
    1. Contains exactly one [1..1] Alerts Section (templateId: 2.16.840.1.113883.10.20.1.2)
  6. SHOULD contain exactly one [1..1] component (CONF-298)
    1. Contains exactly one [1..1] Medications Section (templateId: 2.16.840.1.113883.10.20.1.8)
  7. SHOULD contain exactly one [1..1] component (CONF-388)
    1. Contains exactly one [1..1] Results Section (templateId: 2.16.840.1.113883.10.20.1.14)
  8. SHOULD contain exactly one [1..1] component
    1. Contains exactly one [1..1] Procedures Section (templateId: 2.16.840.1.113883.10.20.1.12)
  9. SHOULD contain exactly one [1..1] component
    1. Contains exactly one [1..1] Encounters Section (templateId: 2.16.840.1.113883.10.20.1.3)
  10. SHOULD contain exactly one [1..1] component
    1. Contains exactly one [1..1] Plan Of Care Section (templateId: 2.16.840.1.113883.10.20.1.10)
  11. SHOULD contain exactly one [1..1] component (CONF-376)
    1. Contains exactly one [1..1] Immunizations Section (templateId: 2.16.840.1.113883.10.20.1.6)
  12. SHOULD contain exactly one [1..1] component (CONF-381)
    1. Contains exactly one [1..1] Vital Signs Section (templateId: 2.16.840.1.113883.10.20.1.16)
  13. SHOULD contain exactly one [1..1] component (CONF-371)
    1. Contains exactly one [1..1] Medical Equipment Section (templateId: 2.16.840.1.113883.10.20.1.7)
  14. SHOULD contain exactly one [1..1] component (CONF-123)
    1. Contains exactly one [1..1] Functional Status Section (templateId: 2.16.840.1.113883.10.20.1.5)
  15. SHOULD contain exactly one [1..1] component (CONF-77)
    1. Contains exactly one [1..1] Advance Directives Section (templateId: 2.16.840.1.113883.10.20.1.1)
  16. SHOULD contain exactly one [1..1] component
    1. Contains exactly one [1..1] Payers Section (templateId: 2.16.840.1.113883.10.20.1.9)
  17. SHALL contain exactly one [1..1] code/@code="34133-9" Summarization of episode note (CodeSystem: 2.16.840.1.113883.6.1 LOINC) (CONF-1)
  18. SHALL contain exactly one [1..1] languageCode (CONF-5)
    1. languageCode has the form nn, or nn-CC. The nn portion SHALL be an ISO-639-1 language code in lower case. The CC portion, if present, SHALL be an ISO-3166 country code in upper case (CONF-6)
  19. MAY contain exactly one [1..1] component (CONF-15)
    1. Contains exactly one [1..1] Purpose Section (templateId: 2.16.840.1.113883.10.20.1.13)
  20. SHALL satisfy: Contains exactly one documentationOf / serviceEvent (CONF-2)
  21. documentationOf / serviceEvent / @classCode SHALL be 'PCPR' (CONF-3)
  22. SHALL satisfy: documentationOf / serviceEvent contains exactly one serviceEvent / effectiveTime / low and exactly one serviveEvent / effectiveTime / high (CONF-4)
  23. SHALL NOT contain templateId / @extension (CONF-8)
  24. SHALL satisfy: effectiveTime is expressed with precision to include seconds (CONF-9)
  25. SHALL satisfy: effectiveTime includes an explicit time zone offset (CONF-10)
  26. SHALL satisfy: Contains one or two recordTarget (CONF-11)
  27. SHOULD satisfy: Contains one or more author / assignedAuthor / assignedPerson and/or author / assignedAuthor / representedOrganization (CONF-12)
  28. If author has an associated representedOrganization with no assignedPerson or assignedAuthoringDevice, then the value for author / assignedAuthor / id / @NullFlavor SHALL be 'NA' (CONF-13)
  29. MAY satisfy: Contains one or more informationRecipient (CONF-14)
  30. The value for component / structuredBody / component / section / entry / @typeCode MAY be 'DRIV' "is derived from" 2.16.840.1.113883.5.1002 ActRelationshipType STATIC, to indicate that the CDA Narrative Block is fully derived from the structured entries. (CONF-28)
  31. A CCD entry SHOULD explicitly reference its corresponding narrative (using the approach defined in CDA Release 2, section 4.3.5.1 <content>). (CONF-29)
  32. A section MAY contain one or more comments, either as a clinical statement or nested under another clinical statement. (CONF-502)
