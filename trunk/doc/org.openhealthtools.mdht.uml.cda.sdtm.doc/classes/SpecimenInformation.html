Specimen Information

[Procedure: templateId 2.16.840.1.113883.10.20.23.59]

The reusable structure carrying specimen information is has, as its starting point, the act of specimen collection. Specimen information is captured as characteristics of the entity playing the specimen role, while specimen condition is recorded as a component act.

  1. SHALL contain exactly one [1..1] @classCode="ACT" Act (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
  2. SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
  3. SHOULD contain zero or one [0..1] effectiveTime
    • The point in time at which the specimen on which the test is based was collected. This may also be a time interval with a start point and a stopping point. [SDTM: LBDTC, LBENDTC, MBDTC, PCDTC, PCENDTC ] Note the point in time, or the start of the specimen collection interval is expected data, while the end point of a collection interval is optional.
  4. SHALL contain exactly one [1..1] code, where the @code SHALL be selected from (CodeSystem: 2.16.840.1.113883.3.26.1.1 NCI Thesaurus)
  5. SHOULD contain zero or one [0..1] specimen

    Captures information regarding the entity - material item - playing the role of specimen.

    1. This specimen SHALL contain zero or one [0..1] @typeCode="SPC"
    2. This specimen SHOULD contain zero or one [0..1] specimenRole
      1. This specimenRole SHALL contain zero or one [0..1] @classCode="SPEC"
      2. This specimenRole MAY contain zero or one [0..1] id

        An identifier for the specimen on which the test result is based. [SDTM: LBREFID, MBREFID, MSREFID, PCREFID]

      3. This specimenRole SHOULD contain zero or one [0..1] specimenEntity
        1. This specimenEntity SHALL contain zero or one [0..1] @classCode="ENT"
        2. This specimenEntity SHALL contain zero or one [0..1] @determinerCode="INSTANCE"
        3. This specimenEntity SHALL contain zero or one [0..1] code

          Used to indicate the specimen type. [SDTM: LBSPEC, PCSPEC, PPSPEC]

  6. MAY contain zero or one [0..1] entryRelationship

    Used to record information regarding the condition of the specimen at the point of testing.

    1. This entryRelationship SHALL contain exactly one [1..1] @typeCode="COMP"
    2. This entryRelationship SHALL contain exactly one [1..1] observation
      1. This observation Contains exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
      2. This observation SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
      3. This observation SHALL contain exactly one [1..1] code, where the @code SHALL be selected from (CodeSystem: 2.16.840.1.113883.3.26.1.1 NCI Thesaurus)

        The value indicates the nature of the observation. It provides information on the specimen condition.

      4. This observation SHALL contain exactly one [1..1] value with data type CE

        Information on the condition of a specimen used for testing. [SDTM: LBSPCCND, MBSPCCND, PCSPCCND]

  7. MAY contain zero or one [0..1] entryRelationship
    • Used to record the study day associated with an activity taking place during a single day.
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Event Study Day (templateId: 2.16.840.1.113883.10.20.23.59)
  8. MAY contain zero or one [0..1] entryRelationship
    • Orients the performance of the test on the specimen with respect to a study defined timepoint.
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Timing Reference (templateId: 2.16.840.1.113883.10.20.23.70)
  9. MAY contain zero or more [0..*] targetSiteCode
    • The body site from which a specimen was collected. {SDTM: MBLOC]
