Study Subject Intervention

[SubstanceAdministration: templateId 2.16.840.1.113883.10.20.23.101]

The template captures investigational, therapeutic and other treatments that are administered to the subject (with some actual or expected physiological effect) either as specified by the study protocol (e.g., "exposure"), coincident with the study assessment period (e.g., "concomitant medications"), or other substances self-administered by the subject (such as alcohol, tobacco, or caffeine). It is included to make allowances for the creation of intervention based domains that are not currently modeled within SEND or SDTM.

  1. SHALL contain exactly one [1..1] @classCode="SBADM" (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
  2. SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
  3. SHOULD contain zero or one [0..1] doseQuantity
    • The amount of a treatment taken at a single administration. The physical quanitity data type is used. Dose units go into the units property of the datatype. [Study Data Tablulation Model: -DOSE, DOSU]
  4. MAY contain zero or one [0..1] effectiveTime
    • Information on when and how oftern the treatment was taken. Possible values include start date, stop date, and duration. Note, only two of these need to be valued, and only two values are supported. [Study Data Tablulation Model: -DOSFRQ, ]
  5. SHALL contain [1..2] id
    • A sponsor supplied reference number for the treatment. A sequence number is required.
  6. MAY contain zero or one [0..1] routeCode
    • The route by which the treatment is administered. [Study Data Tabulation Model: -ROUTE]
  7. MAY contain zero or one [0..1] maxDoseQuantity
    • Information on the total daily dose of the medication or treatment. The structure also accommodates periods other than a day. Note, this attribute can also be used if multiple total dosage amounts need to be reported. [Study Data Tablulation Model: -DOSTOT]
  8. MAY contain zero or one [0..1] text
    • A text description of the intervention or dosing. [Study Data Tablulation Model: -DOSTXT]
  9. MAY contain zero or one [0..1] approachSiteCode with data type CE
    • An indication of the location, e.g. left arm, where the substance was administered. [Study Data Tabulation Model: -LOC]
  10. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Group Identifier (templateId: 2.16.840.1.113883.10.20.23.63)
  11. SHOULD contain zero or one [0..1] consumable
  12. MAY contain zero or one [0..1] entryRelationship
    1. Contains exactly one [1..1] Study Epoch (templateId: 2.16.840.1.113883.10.20.23.71)
  13. MAY contain zero or one [0..1] entryRelationship
    1. This entryRelationship SHALL contain exactly one [1..1] @typeCode="COMP"
    2. This entryRelationship SHALL contain exactly one [1..1] observation
      1. This observation SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
      2. This observation SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
      3. This observation SHALL contain exactly one [1..1] code, where the @code SHALL be selected from (CodeSystem: 2.16.840.1.113883.3.26.1.1 NCI Thesaurus)

        The code value indicates the observation captures an element's order within a trial arm.

      4. This observation SHALL contain zero or more [0..*] value with data type INT

        A value that gives the order of an element within the trial arm. [SDTM: TAETORD]

  14. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Timing Reference (templateId: 2.16.840.1.113883.10.20.23.74)
  15. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Study Day Period (templateId: 2.16.840.1.113883.10.20.23.70)
  16. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Pre-Specified Event (templateId: 2.16.840.1.113883.10.20.23.66)
  17. MAY contain zero or one [0..1] statusCode
    • Used to indicate whether the activity - use of a particular medication or treatment - did occur. Use statusCode "Completed" if the use occurred. [Study Data Tablulation Model: -OCCUR]
  18. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Data Collection (templateId: 2.16.840.1.113883.10.20.23.59)
  19. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Non Performance Reason (templateId: 2.16.840.1.113883.10.20.23.64)
  20. MAY contain zero or more [0..*] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Comment (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  21. SHALL contain exactly one [1..1] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Domain Assignment (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  22. MAY contain zero or more [0..*] entryRelationship
    1. Contains @typeCode="REFR" REFR
    2. Contains exactly one [1..1] Related Record (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  23. MAY contain zero or more [0..*] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Supplemental Value (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  24. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Indication (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  25. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Intended Regimen (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  26. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Dose Adjustment Reason (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  27. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Category (templateId: 2.16.840.1.113883.10.20.23.xxxx)
  28. MAY contain zero or one [0..1] entryRelationship
    1. Contains @typeCode="COMP" COMP
    2. Contains exactly one [1..1] Sub- Category (templateId: 2.16.840.1.113883.10.20.23.xxxx)
