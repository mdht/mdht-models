Supplemental Value

[Observation: templateId 2.16.840.1.113883.10.20.23.xxxx]

A structure to allow the addition of supplementation information to a particular data item.

  1. SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
  2. SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
  3. SHALL contain exactly one [1..1] code
    • A code that indicates the type of information being provided. [SDTM: QNAM, QLABEL], [SEND: QNAM, QLABEL]
  4. SHALL contain exactly one [1..1] value
    • The data value that is being provided. Note, since data type of the data is not predefined, it must be indicated in the instance. [SDTM: QVAL], [SEND: QVAL]
  5. SHALL contain exactly one [1..1] entryRelationship
    1. This entryRelationship SHALL contain exactly one [1..1] @typeCode
    2. This entryRelationship SHALL contain zero or one [0..1] observation
      1. This observation SHALL contain exactly one [1..1] @classCode="OBS" Observation (CodeSystem: 2.16.840.1.113883.5.6 HL7ActClass)
      2. This observation SHALL contain exactly one [1..1] @moodCode="EVN" Event (CodeSystem: 2.16.840.1.113883.5.1001 HL7ActMood)
      3. This observation SHALL contain exactly one [1..1] code

        The code value "ASSERTION" is used.

      4. This observation SHALL contain exactly one [1..1] value with data type CD

        An indication of the origin if the data. Example values include CRF, ASSIGNED, Derived. [SDTM: QORIG], [SEND: QORIG]

  6. MAY contain zero or one [0..1] participant
    • Identification of the party responsible for (author) the supplemental information.
    1. Contains exactly one [1..1] Study Finding Evaluator (templateId: 2.16.840.1.113883.10.20.23.72)
